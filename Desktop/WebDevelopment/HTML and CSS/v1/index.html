The Building Blocks of Every Webpage

HTML structures the content of the web. Tags define meaning, elements create hierarchy, and attributes configure behavior.

1. The 10 Most Useful HTML Tags

These tags appear in virtually every project.

  1. <div> — All-purpose block container for layout and grouping
  2. <p> — Paragraph of body text
  3. <a> — Anchor tag for hyperlinks, uses the href attribute
  4. <img> — Embeds an image, self-closing, always use alt
  5. <ul> / <li> — Unordered list and list items
  6. <section> — Semantic grouping of related content
  7. <input> — Form field, behavior changes with the type attribute
  8. <button> — Clickable control for actions
  9. <span> — Inline container for styling part of a text
  10. <h1> to <h6> — Six levels of headings for hierarchy

2. Most Important SEO Tags

These tags go in the head of your page and tell search engines what your page is about.

In the <head> — invisible to users

  • <title> — The clickable headline shown in Google results. Most important SEO tag. Keep under 60 characters.
  • <meta name="description"> — The summary shown under the title in results. Boosts click-through rate. Keep under 160 characters.
  • <meta name="viewport"> — Enables proper mobile display. Google uses mobile-first indexing.
  • <meta name="robots"> — Tells Google whether to index the page and follow its links.
  • <link rel="canonical"> — Points Google to the real URL when duplicate pages exist.
  • <meta property="og:*"> — Open Graph tags, control how the page looks when shared on social media.
  • <meta name="twitter:*"> — Same as above but specifically for X / Twitter.

In the <body> — visible on the page

  • <h1> to <h6> — Heading hierarchy. One h1 per page only. Google uses these to understand content structure.
  • <img alt="..."> — Alt text on images. Google reads the alt attribute to understand what images show.

One to avoid

<meta name="keywords"> — Google and Bing both ignore it completely. Safe to leave out.

4. Anatomy of an HTML Document

Every HTML file follows the same essential skeleton: a doctype declaration, a root <html> element, a <head> for metadata, and a <body> for visible content.

<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="UTF-8" />
    <title>My Webpage</title>
  </head>
  <body>
    <h1>Hello, World!</h1>
    <p>Welcome to my site.</p>
  </body>
</html>

5. Page Structure & Semantic HTML

Semantic tags give your markup meaning. They tell browsers, search engines, and screen readers what each piece of content actually is.

  • <nav> — Navigation links
  • <header> — Introductory content or logo area
  • <main> — The primary content (only one per page)
  • <section> — A thematic group of content
  • <article> — Self-contained content like a blog post
  • <aside> — Sidebar or supplementary content
  • <footer> — Bottom of page content
