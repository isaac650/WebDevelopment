The Building Blocks of Every Webpage

HTML structures the content of the web. Tags define meaning, elements create hierarchy, and attributes configure behavior.

1. The 10 Most Useful HTML Tags

These tags appear in virtually every project.

  1. <div> — All-purpose block container for layout and grouping
  2. <p> — Paragraph of body text
  3. <a> — Anchor tag for hyperlinks, uses the href attribute
  4. <img> — Embeds an image, self-closing, always use alt
  5. <ul> / <li> — Unordered list and list items
  6. <section> — Semantic grouping of related content
  7. <input> — Form field, behavior changes with the type attribute
  8. <button> — Clickable control for actions
  9. <span> — Inline container for styling part of a text
  10. <h1> to <h6> — Six levels of headings for hierarchy

2. Most Important SEO Tags

  • <title> — Clickable headline in Google results. Keep under 60 characters.
  • <meta name="description"> — Summary shown under the title. Keep under 160 characters.
  • <meta name="viewport"> — Required for mobile display and Google's mobile-first indexing.
  • <meta name="robots"> — Controls whether Google indexes the page and follows its links.
  • <link rel="canonical"> — Prevents duplicate content by pointing to the preferred URL.
  • <meta property="og:*"> — Controls how the page appears when shared on social media.
  • <h1> to <h6> — One h1 per page. Google uses heading hierarchy to understand structure.
  • <img alt="..."> — Google reads alt text to understand image content.

4. Anatomy of an HTML Document

Every page needs a doctype, an <html> root, a <head> for metadata, and a <body> for content.

<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="UTF-8" />
    <title>My Webpage</title>
  </head>
  <body>
    <h1>Hello, World!</h1>
    <p>Welcome to my site.</p>
  </body>
</html>

5. Page Structure & Semantic HTML

Semantic tags give your markup meaning. They tell browsers, search engines, and screen readers what each piece of content actually is.

  • <nav> — Navigation links
  • <header> — Introductory content or logo area
  • <main> — The primary content (only one per page)
  • <section> — A thematic group of content
  • <article> — Self-contained content like a blog post
  • <aside> — Sidebar or supplementary content
  • <footer> — Bottom of page content

8. Font Sizes — Absolute & Relative Units

Absolute units are fixed. Relative units scale based on context.

  • 12px — absolute Use for: fine print, UI labels, anything that must never scale.
  • 1.5cm — absolute Use for: print layouts like CVs or invoices where physical size matters.
  • 1em — relative to parent Use for: component internals — padding and text that should scale together.
  • 1.5rem — relative to root Use for: page-wide type sizes — consistent across all nesting levels.
  • 120% — relative to parent Use for: sub-headings that should always read slightly larger than body text.
  • 3vw — relative to viewport width Use for: hero headlines that fluidly fill the screen on any device.
